이번 릴리즈에서는 서비스의 보안성과 안정성을 높이고, 개발 효율성을 개선하기 위한 대규모 리팩토링을 진행했습니다. 데이터베이스 접속 정보를 안전하게 암호화하고, 파편화되어 있던 UI 코드를 모듈화하여 유지보수가 쉬운 구조로 개선했습니다. 또한, 동시성 환경에서도 데이터가 정확하게 처리될 수 있도록 로직을 고도화했습니다.

### 1. 보안 강화: 민감 정보의 안전한 관리
설정 파일에 노출되어 있던 데이터베이스 URL과 비밀번호 등 민감한 정보를 보호하기 위해 Jasypt 암호화 라이브러리를 도입했습니다. 현대적인 보안 표준인 `PBEWITHHMACSHA512ANDAES_256` 알고리즘을 적용하여 보안 강도를 높였으며, 암호화 키는 환경 변수를 통해 외부에서 주입받도록 설계하여 코드 유출 시의 위험을 원천 차단했습니다.

### 2. 아키텍처 개선: UI 컴포넌트 모듈화
중복으로 작성되던 게시물 관련 UI 코드를 Thymeleaf Fragment 기능을 활용해 컴포넌트 단위로 분리했습니다.
*   **컴포넌트 중심 개발**: 게시물 목록 아이템, 상세 보기 본문, 하단 버튼 영역 등을 각각의 프래그먼트로 모듈화하여 코드 재사용성을 높였습니다.
*   **JS 로직 통합**: 각 페이지에 흩어져 있던 좋아요 처리 로직을 외부 자바스크립트 파일(`like.js`)로 통합하여 비즈니스 로직의 일관성을 확보했습니다.

### 3. 안정성 및 성능 최적화: 동시성 제어와 효율적 조회
빈번하게 발생하는 데이터 갱신 상황에서도 데이터의 무결성을 유지할 수 있도록 개선했습니다.
*   **원자적 카운트 업데이트**: 좋아요 수나 리트리(가지치기) 수 증가 시, 데이터베이스 레벨에서 직접 연산을 수행하는 `@Modifying` 쿼리를 적용했습니다. 이를 통해 여러 사용자가 동시에 요청을 보내더라도 데이터 유실 없이 정확한 수치가 반영됩니다.
*   **쿼리 최적화**: 단순히 수치 정보만 필요한 경우 엔티티 전체를 조회하지 않고 필요한 필드만 선택적으로 가져오도록 개선하여 데이터베이스 부하를 줄였습니다.
*   **비동기 페이징**: 게시물 상세 페이지에서 해당 글을 바탕으로 생성된 자식 게시물들을 '더보기' 방식으로 비동기 로딩할 수 있는 기능을 추가했습니다.

### 4. 배포 환경 고도화
클라우드 배포 환경인 Railway와의 연동을 강화했습니다. 암호화 마스터 키를 배포 환경의 변수(Variable)로 관리하도록 설정하여, 로컬 개발 환경과 실제 운영 환경 간의 설정 분리를 명확히 하고 보안성을 유지하며 배포할 수 있는 기반을 마련했습니다.
